저번에 라이프 사이클 관련 영상을 보고 우아한테크코스 테코톡에 도움이 되는 영상들이 많은 것 같아 둘러보다 `타입스크립스<제네릭>`이 눈에 들어왔다.

### 제네릭이란?

객체, 함수, 변수의 타입을 미리 정하지 앟고 필요할 때 타입을 유현하게 정할 수 있도록 하는 타입스크립트의 기능

### 함수에 활용

```
function 함수<T>(...args:T):T
```

### 변수에 활용

```
type Nullish<T> = T|null|undefined
```

### 객체에 활용

(에러 발생)

```
type Pick<T,K> = {
 [key in K]:T[key]
}
```

### 제네틱 타입 제한

T,K 타입 매개변수는 모든 타입이 될 수 있다. 즉, any를 사용하는 것과 같다.

```
type Pick<T,K extends keyof T> = {
 [key in K] :T[key]
}
```

### 제네릭, 꼭 써야만 하는가?

- 반복을 줄일 수 있는 관점에서 좋음.
- 코드가 빠르게 복잡해질 수 있음.
- 제네릭을 사용할 때는 무엇을 위해 사용하는지 명확히 해야 함.
- 타입을 외부에서 결정할 수 있도록 유연성을 제공
- extends 키워드로 타입 안정성을 보장
- 타입을 재사용하기 위한 것
