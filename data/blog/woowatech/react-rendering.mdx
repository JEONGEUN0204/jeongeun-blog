### 리액트의 렌더링

컴포넌트가 props와 state를 통해 UI를 어떻게 구성할지 컴포넌트에게 요청하는 작업

1. 트리거(by props,state)
   - `리렌더링` 발생
     - useState의 setter가 실행되는 경우
     - useReducder의 dispatch가 실행되는 경우
     - key props가 변경되는 경우
2. 리액트의 렌더링(컴포넌트에게 요청) : 이 과정에서 가상 DOM이 생성되거나 업데이트 된다.
   - 렌더 단계: 실제 DOM에 반영할 변경 사항을 파악하는 단계
     - JSX코드는 브라우저가 이해할 수 없기 때문에 JS 코드로 변경해야함.
     - JSX가 빌드 시 babel이나 webpack과 같은 도구를 사용해 React.createElement를 호출하여 React Element 객체 생성
     - 이렇게 생성된 수많은 객체 집합들은 트리 형태로 결합되어 가상 DOM을 만듦
     - `리렌더링`의 경우 가상 DOM 비교가 이루어진다. (diffing 알고리즘: 두 가상 DOM 트리 간의 차이점을 효율적으로 찾아내어, 실제로 변경된 부분만을 선택적으로 업데이트한느 과정)
   - 커밋 단계: 실제 DOM에 반영하는 단계
     - 리렌더링의 경우 batch update (실제 DOM을 한 번만 업데이트)
3. 브라우저 렌더링(UI 구성)

diffing 알고리즘 + batch update = 재조정(Reconciliation)

### 가상DOM

- 리액트가 UI의 상태를 메모리상에서 관리하는 가상적인 구조
- 실제 DOM을 추상화 한 상태
- 브라우저 화면과는 분리된 자바스크립트 객체
- 접근/수정 시 실제 브라우저 화면에 영향 X
- prev 가상 DOM, next 가상 DOM 두 개의 가상 DOM 만 사용
- DOM 변경이 발생하면 웹 브라우저는 스타일과 레이아웃을 다시 계산하고 페이지를 다시 그려야하는데, 이때 가상 DOM을 사용하여 실제 DOM에 대한 직접 변경 수를 줄일 수 있다.
- 가상 DOM을 통해 변경된 부분말 선택적으로 업데이트하여 사용자에게 빠르고 매끄러운 UI경험을 제공한다.
